تعریف کلی درباره ساخار نست رو میگم :
ساختار ماژول های نست از 3 تا فایل اصلی ایجاد شده

Module: 
    از اسمش معلومه که ماژول اصلی هستش
    انگار که تمام اتفاقات رو کنترل میکنه و بقیه فابل ها دارن زیر نظر این کار میکنن

Controller: 
    این فایل هم از اسمش پیداست که کنترلر پروژست و تمام رکوئست ها اینجا هندل میشه

Service: 
    این فایل هم باز با توجه به اسمش داره یه خدمتی رو به کنترلر ارائه میده
    مثل ارتباط با دیتابیس و چیزای دیگه

به صورت خلاصه میشن همون مدل و کنترلر



برای افزایش سرعت کامپایل از دوتا ابزار جدید استفاده میکنیم
    npm i -D @swc/cli @swc/core

و فایل nest-cli.json رو به مقدار زیر تغییر میدیم
    {
    "$schema": "https://json.schemastore.org/nest-cli",
    "collection": "@nestjs/schematics",
    "sourceRoot": "src",
    "compilerOptions": {
        "deleteOutDir": true,

        این دوتا اضافه شد
        "builder": "swc",
        "typeCheck": true
    }
    }
